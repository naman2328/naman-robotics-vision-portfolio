Why Vision Systems Fail in Factories

Real production environments introduce glare, vibration, inconsistent part positioning, and timing constraints. Naive vision pipelines that work in controlled conditions often fail without deterministic logic, validation, and traceability.

Projects & Hands-On Work

These projects reflect my learning journey in robotics and computer vision. The focus was on understanding real hardware behavior, failure modes, and how software logic connects with physical systems.

OCR-Based Vision Inspection (Jetson Orin Nano)

Built an offline OCR inspection setup using PaddleOCR to study text detection and recognition on real camera images. Emphasis was placed on ROI selection, confidence handling, and OK / NG decision logic.

What I learned: OCR limitations, lighting impact, ROI importance, inspection logic.

Tools: PaddleOCR · OpenCV · Python · Jetson Orin Nano

Vision-Guided Pick & Place (ROS 2 + Dobot)

Implemented a basic vision-guided pick-and-place system using ROS 2. Vision processing and robot control were separated into independent nodes to understand ROS communication and real robot behavior.

What I learned: ROS 2 fundamentals, vision-to-motion flow, hardware debugging.

Tools: ROS 2 · OpenCV · Python · Dobot Magician

Custom 3D-Printed Robotic Arm

Designed and assembled a small robotic arm to understand mechanical structure, servo control, and basic motion sequencing. This project exposed real-world calibration and hardware issues.

What I learned: mechanical assembly, servo control, practical debugging.

Tools: Python · Servo Motors · Mechanical Design

Omnidirectional Mobile Robot (Mecanum Wheels)

Built an omnidirectional mobile robot to explore holonomic motion. Implemented wheel kinematics and tested lateral and rotational movement.

What I learned: wheel kinematics, motor control, motion testing.

Tools: Python · Motor Drivers · Mecanum Kinematics

Failure-First Engineering Approach

  • Glare & reflections → ROI constraints & preprocessing
  • False positives → validation rules & fuzzy matching
  • Latency issues → region-only processing
  • Unstable inputs → deterministic state handling
