Industrial Vision Inspection System

Camera-based OCR and visual inspection system designed for industrial environments where reliability, repeatability, and deterministic decisions matter more than raw accuracy metrics.

Problem Statement

In manufacturing environments, visual inspection must operate continuously under non-ideal conditions such as variable lighting, glare, motion blur, and part misalignment.

Manual inspection is slow and inconsistent, while naive computer-vision pipelines often fail due to false positives, unstable OCR results, and lack of validation logic. A usable industrial vision system must therefore behave like deterministic software, not a one-off demo.

System Overview

The inspection system follows a fixed and predictable pipeline:

  • Industrial camera captures the frame
  • Region of Interest (ROI) is locked to avoid irrelevant areas
  • OCR / visual inspection runs only inside the ROI
  • Detected output is validated against expected patterns
  • Final OK / NG decision is generated
  • Inspection result and timestamp are logged

This flow ensures stable performance and predictable behavior on a production line.

What I Built

  • Designed ROI-based inspection to minimize false detections
  • Integrated PaddleOCR for text recognition inside constrained regions
  • Implemented validation logic instead of trusting raw OCR output
  • Built OK / NG decision logic suitable for automation systems
  • Added timestamped logging for inspection traceability

The focus was not on maximizing OCR accuracy in isolation, but on building a complete inspection pipeline that behaves reliably in real conditions.

What Failed and How It Was Fixed

  • OCR instability due to glare: Reduced inspection area to a fixed ROI and avoided reflective regions.
  • False positives from background text: Introduced strict validation rules and rejection logic.
  • Frame drops and latency: Limited processing to only required regions instead of full-frame OCR.
  • Unreliable OCR strings: Treated OCR output as an input signal, not as a final truth.

These changes shifted the system from a fragile demo to a predictable, production-oriented inspection pipeline.

Why This Is Production-Oriented

  • Deterministic OK / NG decision logic
  • Clear separation between detection and validation
  • Continuous operation mindset
  • Traceable inspection results
  • Designed for integration with PLC or automation controllers
