Industrial Computer Vision

Vision Inspection System

A deterministic, factory-oriented vision inspection system designed for noisy production environments — not lab demos.

Why Vision Fails in Factories

Most computer-vision pipelines collapse on the shop floor due to glare, motion blur, alignment shifts, and uncontrolled backgrounds.

Industrial systems demand repeatability, predictable timing, and clear decisions — not just accuracy scores.

System Architecture

Camera captures frame
ROI locked
OCR / inspection inside ROI
Validation logic
OK / NG decision
Logs + timestamps

What I Engineered

  • ROI-restricted inspection to suppress false positives
  • PaddleOCR integrated as a signal source, not final truth
  • Strict validation rules for production stability
  • Deterministic OK / NG decision pipeline
  • Traceable logging for audits and debugging

Failure-First Engineering

  • Glare → OCR instability → ROI constraints
  • Background text → false positives → validation filters
  • Latency spikes → full frame OCR → region-only processing

These failures shaped the system more than initial success.

Why This Is Production-Oriented

  • Single decision per cycle
  • Fail-safe logic on uncertainty
  • Continuous operation mindset
  • PLC-compatible OK / NG signaling
