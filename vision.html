Vision Inspection & Robotics Systems

Factory-oriented vision and inspection systems designed for unstable lighting, motion, alignment errors, and real-world production constraints — not laboratory demos or offline-only pipelines.

Why Vision Systems Fail in Factories

Real production environments introduce glare, vibration, inconsistent part positioning, and timing constraints. Naive vision pipelines that work in controlled conditions often fail without deterministic logic and validation.

Projects Implemented

The following projects were developed using real hardware and production-style workflows, with emphasis on robustness, validation, and system integration.

Industrial OCR Inspection System (Jetson Orin Nano)

Designed and deployed an offline OCR inspection pipeline using PaddleOCR v3, capable of operating under factory lighting and alignment variations. Implemented ROI-based processing, fuzzy text validation, confidence scoring, and deterministic OK / NG decision logic.

Tools: PaddleOCR · OpenCV · Python · Jetson Orin Nano

Vision-Guided Pick & Place (ROS 2 + Dobot Magician)

Semi-autonomous pick-and-place system built using ROS 2 (Humble) with dedicated nodes for vision processing, decision logic, and robotic arm control. Implemented HSV-based color detection using a USB camera and integrated perception output with real hardware motion.

Tools: ROS 2 · OpenCV · Python · Dobot Magician

Custom 3D-Printed Robotic Arm

Designed, assembled, and programmed a custom multi-DOF robotic arm using 3D-printed components. Implemented servo-based joint control, motion sequencing, calibration routines, and hardware debugging in Python.

Tools: Python · Servo Control · Mechanical Design·Computer vision

Omnidirectional Mobile Robot (Mecanum Drive)

Designed and developed an omnidirectional mobile robot using mecanum wheels for holonomic motion. Implemented independent wheel velocity control and tested motion behaviors such as lateral translation and in-place rotation.

Tools: Python · Motor Drivers · Kinematics · Embedded Control

Failure-First Engineering Approach

  • Glare & reflections → ROI constraints and preprocessing
  • False positives → validation rules & fuzzy matching
  • Latency issues → region-only processing
  • Unstable inputs → deterministic state handling
